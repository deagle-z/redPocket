# 试玩功能实施计划

## Summary

为登录用户新增独立试玩体系：新注册账号获得 `trial_balance=10000` 试玩额度；正式余额、正式红包、正式机器人数据不混用。试玩完整复刻当前红包玩法，但使用独立试玩数据表、独立 H5 试玩页面、独立后台试玩机器人管理页面。

## Key Changes

- 后端用户模型 `tg_user` 增加 `trial_balance decimal(20,2) not null default 10000`。
- 邮箱注册和手机号注册创建新用户时显式写入 `trial_balance=10000`；现有用户默认按迁移默认值处理，不批量补发。
- 新增试玩专用表：
  - `trial_lucky_money`
  - `trial_lucky_money_item`
  - `trial_lucky_history`
  - `trial_bot_user`
  - `trial_cash_history`
- 新增试玩服务层，复用正式玩法算法思想，但所有读写都指向试玩表和 `tg_user.trial_balance`。
- 新增 H5 页面 `/trial`，展示试玩额度、发包、抢包、雷号/单双玩法。
- 新增后台页面“试玩机器人”和“试玩记录”。

## Implementation Steps

- 后端模型与迁移：
  - 修改 `core/pojo/tg_user.go`，增加 `TrialBalance`。
  - 新增 `core/pojo/trial_lucky_money.go`、`trial_lucky_money_item.go`、`trial_lucky_history.go`、`trial_bot_user.go`、`trial_cash_history.go`。
  - 在 AutoMigrate 注册新表。
- 后端注册初始化：
  - 修改 `RegisterTgByEmail`、`RegisterTgByPhone`，新用户 `TrialBalance=10000`。
  - Telegram 登录自动创建的新用户同样初始化 `TrialBalance=10000`。
- 后端试玩玩法：
  - 新增 `core/services/trial_lucky_money_service.go`，实现试玩发包、抢包、机器人自动抢、过期退款。
  - 新增 `core/repository/trial_lucky.go`，所有试玩查询严格限定试玩表。
  - 新增 `core/api/trial_lucky_api.go` 和 `core/api/trial_bot_api.go`。
  - 在 `core/common/web_routes.go` 注册 app auth 和 admin 路由。
- H5：
  - 新增 `RedPocketH5/src/pages/trial/index.vue`。
  - 新增 `RedPocketH5/src/api/trial.ts`。
  - 新增 `/trial` 独立页面，不加入底部主导航。
- 后台：
  - 新增 `pure-admin-thin/src/api/trialBot.ts`、`trialLucky.ts`。
  - 新增 `pure-admin-thin/src/views/system/trial_bot/index.vue`。
  - 新增 `pure-admin-thin/src/views/system/trial_lucky/index.vue`。
  - 菜单通过现有菜单系统配置，组件路径分别为对应 view 路径。

## Test Plan

- Go 单元测试：
  - 新注册邮箱用户 `trial_balance=10000`。
  - 新注册手机号用户 `trial_balance=10000`。
  - 试玩发包只扣 `trial_balance`，不影响 `balance/gift_amount/recharge_amount`。
  - 试玩抢包只写 `trial_lucky_history/trial_cash_history`，不写正式 `lucky_history/cash_history`。
  - 试玩机器人只从 `trial_bot_user` 选择，不读取 `tg_user.is_bot`。
  - 试玩红包过期退款只回退 `trial_balance`。
- API 测试：
  - 未登录访问试玩发包/抢包返回未授权。
  - 登录用户可获取试玩额度、发包、抢包、查看历史。
  - 后台可批量创建试玩机器人、启停、删除。
- 前端验证：
  - `/trial` 页面能加载试玩红包、显示试玩额度、发包后额度减少、抢包后额度变化。
  - 正式红包页和正式钱包页不显示试玩流水。
  - 后台试玩机器人页面列表、批量创建、批量修改、状态切换可用。

## Assumptions

- 试玩只面向登录用户，不支持游客模式。
- 试玩完整复刻现有红包玩法，包括雷号和单双。
- 新号初始试玩额度固定为 `10000`，本期不做后台配置项。
- 试玩额度不能提现、不能充值、不能转正式余额、不能参与 VIP/返佣/邀请/平台盈利统计。
